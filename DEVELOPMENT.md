# Development Guide

## Prerequisites

- **Node.js** 18+
- **Xcode** 15+ with iOS 16+ simulator
- **CocoaPods** — `brew install cocoapods`
- An [Apple Developer account](https://developer.apple.com) is not required for simulator testing, but you do need an Apple ID signed in to the simulator for Sign in with Apple

## Repo layout

```
capacitor-clerk/       ← the library (this package)
└── example-app/       ← Capacitor demo app (references library via file:..)
    ├── ios/           ← Xcode project (generated by Capacitor)
    └── src/           ← React source
```

The example-app declares `"capacitor-clerk": "file:.."` in its `package.json`. This means it reads from the library's `dist/` folder, so **you must build the library before your changes appear in the app**.

## Setup

### 1. Install library dependencies

```bash
npm install
```

### 2. Build the library

```bash
npm run build
```

### 3. Set up the example-app

```bash
cd example-app
cp .env.example .env
# Edit .env and add your Clerk publishable key:
# VITE_CLERK_PUBLISHABLE_KEY=pk_test_...
npm install
```

### 4. Install iOS native dependencies

```bash
npx cap sync ios
```

This copies the web build into the Xcode project and installs CocoaPods. Run it again any time you add or update a Capacitor plugin.

## Development loop

### Iterating on library code

Any change to `src/` in the root requires a rebuild before the example-app picks it up:

```bash
# From repo root
npm run build

# Then in example-app, rebuild web and sync native
cd example-app
npm run build
npx cap sync ios
```

For faster TypeScript feedback while editing the library, run the compiler in watch mode in a separate terminal:

```bash
# From repo root — re-compiles on save, but does not run rollup
npm run watch
```

You still need a full `npm run build` (which runs rollup) before syncing to the app.

### Live-reload on simulator (recommended for UI iteration)

Capacitor supports live reload via Vite's dev server. With this setup the simulator loads your React code from the local dev server, so React changes hot-reload instantly without a sync cycle. Native plugin changes (Swift/Kotlin) still require a full rebuild.

```bash
# Terminal 1 — start Vite dev server
cd example-app && npm run start

# Terminal 2 — run on simulator with live reload
cd example-app && npx cap run ios --livereload --external
```

`--external` makes the simulator use your Mac's LAN IP instead of `localhost` so the native WebView can reach the dev server.

> If prompted to choose a simulator, pick any iOS 16+ device. iPhone 16 Pro is a good default.

### Open in Xcode (when you need the native debugger)

```bash
cd example-app && npx cap open ios
```

Then press the Run button in Xcode. Use the Xcode console for native crash logs and Swift breakpoints.

## Testing specific flows

### Sign in with Apple

Requires an Apple ID configured on the simulator:

1. Open the simulator
2. Go to **Settings > Sign in to your iPhone** and sign in with a personal Apple ID (not a work/school account)
3. Tap "Sign in with Apple" in the example-app

The native Apple sheet will appear. On first use Apple may ask for two-factor confirmation on your iPhone.

### Google / OAuth SSO (deep-link flow)

The SSO flow redirects back to `capacitorclerk://sso-callback`. You can fire a fake callback to test the deep-link handler without going through a real OAuth provider:

```bash
xcrun simctl openurl booted "capacitorclerk://sso-callback?rotating_token_nonce=test"
```

Replace `test` with a real nonce from a Clerk-initiated flow if you need the full round-trip.

### Running tests (no simulator needed)

```bash
# From repo root
npm run test          # run once
npm run test:watch    # watch mode
```

## Rebuilding the native project from scratch

If you get unexpected native errors or after updating `@capacitor/ios`:

```bash
cd example-app
rm -rf ios/App/Pods ios/App/Podfile.lock
npx cap sync ios
```

> Any change to `Info.plist` or `AndroidManifest.xml` requires a **full Xcode rebuild** (Product > Clean Build Folder), not just `cap sync`.
